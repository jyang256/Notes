CLOUD is everywhere. It's used by everyone

what is cloud computing?

cloud computing is on demand computing resources, delivered to you over the internet
    It's a computer service you traditionally did local, now performed remotely, across the internet.
  • clouds are elastic - virtually unlimited space
  • SCALABLE... in case you dont want to pay as much
  • METETERED - pay for what you need
  • cloud is SELF SERVICE- no need or almost no need for IT experts
  • applications are not running locally - running In THE CLOUD
  • Think Google's dumb game console - games run on the cloud
  • cloud computing - only pay for waht you need, fast access to applications
  • may eliminate the need for local IT staff since apps are maintained on the cloud
  • potential to LOWER COSTS
  • self-service
  • IT staff can focus on the business

risks of cloud computing

  • You are placing your trust in the cloud provider - susceptible to slowdowns in the cloud
  • potential for data loss - need multiple backups, including other clouds
  • potential to slow access to your data
  • since cloud is remote, you may need to pay attention to laws and stuff
  • potential loss of customization

IAAS - infrastructure as a service

    cirtualization - the logical division of physical computer resources
    VMWare's ESX sever popularized virtualization
    VMWare fusion allows you to run any OS on their remote servers
    Hypervisor - where the virtual machine runs on
    operating system and applications are loaded in the guest on a virtual host
    cirtualliation isnt necessarily cloud computing, but virtualization makes cloud computing possible
    a single server can run several virtual workspaces
    containeer - OS level virtualization
    containers could run inside a virtual machine
    docker is a container
    a container can have a webserver inside. hence a "localhost"
  • private-hybrid-public clouds exist
  • private clouds are run locally
  • public clouds - virtual machines
  • hybrid cloud - connecting a private cloud to a public cloud to work together example: separating applications between public and private
  • enterprise cloud
  • virtualization - scalability/elastic computing
  • virtualization is required for cloud computing
  • resource sharing,, load balancing, high availability, ability to clone, portability
  • private cloud versus virtualization
  • private cloud provides:
  • abstraction of underlying infrasturcture layer, secure multi tenancy, self serivece portal, catalog of applications, chargeback/showback, potential to burst to a hybrid cloud
  • infrastucture as a service (IaaS) pricing - subscription based, pay for what you need
  • on-demand, spot instances, reserved instances - pricing models of IaaS
  • Amazon EC2 is too fast for other companies
  • Service level agreement (SLA): Deifnes what level of performance and availabitily the IaaS provider will provide you
  • For example, EC2 guarantees 99.9% service uptime
  • some companies have greenfield deployments - uncommon
  • consider costs of cloud

Security in the cloud/IaaS

breaches, hacks, many are concerned with cloud security
  • encryption
  • concerns - privacy, permanence
  • unless you have managed securtiy, security is relied on you and the cloud provider
  • AWS has many services related to security, built in
  • encryption - converting info into ciphertext. Can only be unlocked with a key.
  • compliance - many companies have regulatory requirements. Also there are laws, policies, and rules
  • be aware of vulnerabilities. Mitigation includes keeping systems up to date, knowing who performs security scans/tests/maintenance
  • you can report vulnerabilities depnding on what service you are using
